Preface

This is a very minimalistic web project: a static site. It consists of descriptions of two web app deployment methods: manual integration and auto sync to Microsoft Azure AppService with Azure CLI.

Azure CLI deployment

Common steps

Preparation

Azure CLI must be installed locally in order to use the scripts referenced below. Git is also required.

If you aren't logged into your Azure account with Azure CLI, do it first: az login. Then clone the !gh vasily-blinkov/minimum-minimorum repository.

Filling parameters

Set yours values for the next fields in the deployment/parameters.json script:

  1. subscription – subscription I.D.;
  2. resourceGroup – resource group name;
  3. appservicePlan – App Service plan name;
  4. webapp – unique web app name

You may also want to set another value for the githubRepo variable to deploy your web app instead of mine (but I don't mind if you use mine).

Manual integration from external Git

Run the manual-integration.ps1 script. It will create the the app, configure deployment from GitHub with manual integration. This script tells Azure to take the web app sources from GitHub, so you should to push any changes to make the web app reflect them.
